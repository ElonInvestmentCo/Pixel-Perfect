## Problem
The @workspace/mockup-sandbox service fails during the BUILD stage with ERR_PNPM_LOCKFILE_CONFIG_MISMATCH. Railpack runs `pnpm install --frozen-lockfile --prefer-offline`, which fails because the `overrides` configuration in pnpm-workspace.yaml does not match what is recorded in pnpm-lock.yaml.

Build log error:
```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH  Cannot proceed with the frozen installation. The current "overrides" configuration doesn't match the value found in the lockfile
Update your lockfile using "pnpm install --no-frozen-lockfile"
```

## Diagnosis
pnpm-workspace.yaml contains an extensive `overrides` section: CVE fixes for qs (>=6.15.2), postcss (>=8.5.10), and uuid (>=11.1.1); platform-specific binary exclusions for esbuild, rollup, lightningcss, @tailwindcss/oxide, and @expo/ngrok-bin; and an esbuild pin to 0.27.3. These overrides were added or modified after the last time pnpm-lock.yaml was generated, so the lockfile is stale. The --frozen-lockfile flag used by Railpack correctly prevents automatic lockfile updates, causing the build to fail. The triggering commit (041d89dc) only changed auth routes and .replit config, confirming the mismatch pre-existed.

## Fix
This fix cannot be performed by a code agent because it requires running pnpm install to regenerate the lockfile binary.

1. In the repo root, run:
   ```
   pnpm install --no-frozen-lockfile
   ```
2. This regenerates pnpm-lock.yaml to match the current overrides in pnpm-workspace.yaml.
3. Commit and push the updated lockfile:
   ```
   git add pnpm-lock.yaml
   git commit -m "chore: regenerate lockfile to match workspace overrides"
   git push origin main
   ```
4. The new deployment will pass the `pnpm install --frozen-lockfile` step.
